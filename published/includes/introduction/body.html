SET UP:

Once you are signed up with Plotly, you can setup up the Plotly MATLAB API with the newly added plotlysetup.m function. This function takes your username and API key as input and embeds the Plotly API into your MATLAB toolboxes folder and permanently saves your username and API key into a credentials file within a plotly folder located in your directory. Your MATLAB API key can be located ONLINE by signing in and clicking on the settings tab located under your username in the right hand corner of the browser.

>> SETUP THE PLOTLY API ¶

In [25]: 
%%matlab
plotlysetup('matlab_user_guide','7jd7qa2s10');

>> plotlysetup.m

Setup the Plotly MATLAB API

The plotlysetup.m function removes the need to have to add the Plotly MATLAB API to your search path everytime a new MATLAB session is started. It also automatically saves your user_name , api_key , and endpoint_domain (optional) , removing the need to sign in everytime a new MATLAB session is started.

[INPUT]:

>> plotlysetup( 'user_name' , 'api_key' )

>> plotlysetup( 'user_name' , 'api_key' ,end_domain)


[WHERE]:

username - your user_name [string]

api_key - your api_key [string]

end_domain - enpoint domain name [struct]{end_domain.plotly_rest_url = https://plot.ly }


The Plotly MATLAB API is saved in the MATLAB root toolbox directory. If the Plotly MATLAB API already exists in the toolbox directory, the user is prompted for overwrite. The MATLAB root directory startup.m script is modified to add the Plotly MATLAB API toolbox path to the MATLAB search path everytime a new session is started. The plotlysetup.m function calls the saveplotlycredentials.m function and the signin.m function.





>> signup.m

Signup for a free Plotly Account

The signup.m function creates a Plotly account and returns a temporary password and an api_key. This process can also be completed ONLINE .

[INPUT]:

>> resp = signup( 'username' , 'e-mail' )


[WHERE]:

username - your new user name [string]

api_key - your valid email address [string]


[OUTPUT]:

resp - results info of the signup [struct]

resp.[api_key,message,un,temp_pw,error]


[WHERE]:

resp.api_key - your new api key [string]

resp.message - greeting message [string]

resp.un - chosen username [string]

resp.temp_pw - your temporary password [string]

resp.error - potential error messages [string]






>> signin.m

[INPUT]:

>> [un key domain] = signup( 'user_name' , 'api_key' ,end_domain)


[WHERE]:

username - your user_name [string]

api_key - your api_key [string]

end_domain - enpoint domain name [struct]{end_domain.plotly_rest_url = https://plot.ly }


[OUTPUT]:

un - your user_name [string]

key - your api_key [string]

domain - your endpoint domain [struct]{end_domain.plotly_rest_url = https://plot.ly }

>> plotlyhelp.m

Plotly Declarative Syntax Reference

*NEW* to the Plotly MATLAB API is the plotlyhelp.m function, which stores all the essential Plotly data/style/layout declarative syntax references in a nested structure array. Information regarding the Plotly declaritive syntax can be accessed by passing valid fieldnames as input to plotlyhelp.m in the order in which they are nested.

[INPUT]:

>> plotlyhelp(varargin)

>> plotlyref = plotlyhelp(varargin)


[WHERE]:

varargin - comma separated variable [string] arguments.


[OUTPUT]:

plotlyref - Plotly syntax reference [struct].


NOTE: Calling >> plotlyhelp with no input arguments outputs the complete first-level fieldnames of valid Plotly references.The online documentaiton can be accessed directly from MATLAB using >> plotlyhelp('online') .





>> plotly.m

Plot a MATLAB figure object with Plotly

The plotly.m function is the heart of the MATLAB API. It handles the creation of graphs in your Plotly account from MATLAB.

[INPUT]:

>> resp = plotly(x1,y1,x2,y2,..., kwargs)

>> resp= plotly({data1, data2, ...}, kwargs)


[WHERE]:

x1,y1,...xn,yn - data arrays [float]

data1, ..., datan - data/style information [struct]

kwargs - optional keyword arguments [struct]


[KWARGS]:

kwargs.layout - layout of plot [struct]

kwargs.fileopt - plot extension ( 'new' , 'overwrite' , 'append' , 'extend' ) [string] {new}

kwargs.traces - array of traces to be extended [int]

kwargs.world_readable - public/private plot setting ( 'true' / 'false' ) [string] {true}

kwargs.filename - name of plot to be created [string] { 'untitled' }


[OUTPUT]:

resp - results info of the plot [struct]


[WHERE]:

resp.url - address of your newly created Plotly plot [string]

resp.warning - additional warning message [string]

resp.message - additional generic messages [string]

resp.filename - name of plot created [string]

resp.error - any error messages received [string]





fig2plotly.m

Plot a MATLAB figure object with Plotly

*NEW* to the Plotly MATLAB API is the fig2plotly.m function - the future of the MATLAB API. fig2plotly.m allows you to take your MATLAB generated figures and automatically create a Plotly plot, taking care of the conversion from the MATLAB figure object structure to Plotly syntax.

[INPUT]:

>> resp = fig2plotly()

>> resp = fig2plotly(f)

>> resp = fig2plotly(gcf)

>> resp = fig2plotly(f, 'property' ,value, ... )

>> resp = fig2plotly(gcf, 'property' ,value, ... )


[WHERE]:

gcf - root figure object handle [double].

f - root figure object [struct] >> f = get(gcf)


[PROPERTIES]:

'name' - name of the plot [string][ 'untitled' ]

'strip' - use plotly default styling [boolea]{0}

'open' - opens a browser with plot result [boolean]{1}


[OUTPUT]:

resp - results info of the plot [struct]


[WHERE]:

resp.url - address of your newly created Plotly plot [string]

resp.warning - additional warning message [string]

resp.message - additional generic messages [string]

resp.filename - name of plot created [string]

resp.error - any error messages received [string]





>> getplotlyfig.m

Get data, style, and layout from the plots stored online

One of Plotly's secret powers is the ability to translate between MATLAB's structure/cell array syntax and JSON. This allows a smooth transition between the your figures in MATLAB and those stored in your Plotly account. In order to grab the data, style, and layout information from your plots living online, we've introduce a *NEW* funtion into the API: getplotlyfig.m . In fact, we can go one step further: getploylyfig.m lets you grab the data, style and layout of anyone's Plotly plots (as long as they are made public) - not just your own! See a graph you like online? Grab the data, style and layout and make one for yourself.

[INPUT]:

>> plotlyfigure = fig2plotly( 'file_owner' , 'file_id' )


[WHERE]:

file_owner - user name associated with the plot [string].

file_id - number identifier associated with the plot [string]


[OUTPUT]:

plotlyfigure - results info of the plot [struct]


[WHERE]:

plotlyfigure.data - data information associated with the plot [cell].

plotlyfigure.layout - layout information associated with the plot [struct]






>> saveplotlyfig.m

Save your MATLAB figure as an image using Plotly

The *NEW* saveplotlyfig.m allows for conversion from MATLAB figures to PNG using Plotly.

[INPUT]:

>> saveplotlyfig(data, layout, 'filename' )


[WHERE]:


data - data/style of plot [cell]

layout - layout of plot [struct]

filename - name of image to be saved [string]


[OUTPUT]:

PNG image 'filename' saved to your working directory

Currently saveplotlyfig.m supports the creation of PNG images and is in the process of expanding to accomodate other graphics formats.
